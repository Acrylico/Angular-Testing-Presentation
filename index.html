Testing AngularJS with Jasmine

A brief introduction

Created by Holden Profit

What is Jasmine?

"Jasmine is a behavior-driven development framework for testing JavaScript code." You write your test in an easy to read JavaScript file, hit run, and watch everything fail!

So how do I get Jasmine?

By adding it to your bower dependencies!

bower.json


{
  "name": "myApp",
  "version": "0.1.0",
  "dependencies": {
    "jasmine":"latest"
  }
}
And then simply run "bower install" from your gitbash line.

What to test

So let's assume you have an Angular controller as follows: 


myApp.controller('myController', function ($scope) {
	$scope.person = {
		name: "",
		age: 0,
		gender: ""
	};
});

To Begin

You'll need some basic setup for your Angular Module. We'll create a file called "myControllerSpec.js"


describe("myController Tests", function () {
	var scope, controller;
});

Next, a basic test!


describe("myController Tests", function () {
	var scope, controller;

    it("should have a person object", function () {
        expect(scope.person).toBeDefined();
        expect(scope.person).toEqual({
            name: "",
            age: 0,
            gender: ""
		});
    });
	
});

Oh boy! Another bit of setup!

The prior test, however, will fail without telling Jasmine what controller you're actually testing!


describe("myController Tests", function () {
	var scope, controller;

	beforeEach(module("myApp")); // Creates the module "myApp"

	beforeEach(inject(function ($rootScope, $controller) {
		scope = $rootScope.$new(); // Creating a new scope for our controller
		controller = $controller; // The service that instantiates controllers
		// Creates a "myController" controller object
		controller("myController", {$scope: scope}); 
	}));

	afterEach(function () { });
	. . .
});
beforeEach() and afterEach() are setup and teardown functions for your testing, meant to keep things DRY

That's nice...

but I want to test some functionality!

So let's expand the controller a bit... 


myApp.controller('myController', function ($scope) {
	$scope.person = {
		name: "",
		age: 0,
		gender: ""
	};
	
	// Will take some parameters to initialize our person object with
	$scope.initPerson = function (name, age, gender) {
	
	};
});

So we write the test first...


describe("myController Tests", function () {
	. . .
	
    it("should initialize the person object when initPerson() is called", function () {
        var name = "Guy Gardener", age = 30, gender = "male";
		
		scope.initPerson(name, age, gender);

        expect(scope.person).toEqual({
            name: name,
            age: age,
            gender: gender
		});
    });
	
});

Then write the function to pass!


myApp.controller('myController', function ($scope) {
	$scope.person = {. . .};
	
	$scope.initPerson = function (name, age, gender) {
		$scope.person.name = name;
		$scope.person.age = age;
		$scope.person.gender = gender;
	};
});
